# Project Rules

Every new feature implemented in this project MUST include the following components:

1. **Test Coverage**: Minimum 80% line coverage for the new feature.
2. **Integration Tests**: Comprehensive integration tests covering the new functionality.
3. **Evidence Report**: A documented evidence report validating the feature implementation.
4. **Performance Considerations**: Documentation and validation of performance impacts.
5. **Authorization Verification**: Ensure all authorization and role-based access constraints are correctly implemented and verified.
6. **Swagger Verification**: Verify that Swagger/OpenAPI documentation is accurate and reflects the new endpoints, schemas, and security requirements.
